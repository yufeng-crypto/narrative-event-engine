# 叙事引擎挂接Subagent讨论 - 2026-03-03

## 目标
让用户的聊天输入经过叙事引擎处理后返回，实现subagent输出润色。

## 结论

### 不可行原因
- 引擎是Python模块，运行在原型程序进程中
- Provider是Node.js进程
- 两者无法直接互调

### 解决方案
需要给引擎加HTTP服务层（Flask/FastAPI），Provider通过HTTP调用引擎API。

## 技术细节

### 引擎API接口
```python
from engine_llm import create_engine, start_engine, chat, get_state

engine = create_engine()
start_engine(engine, "沈予曦", profile="角色设定")
result = chat(engine, "用户消息")
# result["npc"] = NPC回复
```

### 引擎维护的状态
- 六轴 (axes)
- 动量 (momentum)
- 线程池 (threads)
- 事件池 (event_pool)
- 轮数 (round)
- 对话历史 (history)

### Provider需要做的事
1. 启动时：HTTP POST /api/start 初始化引擎
2. 对话时：HTTP POST /api/chat 传入用户消息，获取NPC回复
3. 直接返回 result["npc"]（不经过上游LLM）

# FTS5搜索问题讨论 - 2026-03-03

## 问题背景
- OpenClaw的memory搜索默认使用向量搜索（sqlite-vec）
- Windows Node.js的SQLite不支持fts5模块
- 用户没有配置embedding模型的API Key

## 讨论结论

### 现状
- `vector ready` = 向量搜索环境就绪（sqlite-vec扩展可用）
- 但没有配置embedding API Key → 向量搜索实际不可用
- fts5不可用 → Windows限制

### 问题分析
系统优先尝试向量搜索（provider: auto → 选了gemini），失败后没有回退到fts5

### 解决方案
在openclaw.json中配置禁用向量搜索，强制使用fts5：

```json
{
  "agents": {
    "defaults": {
      "memory": {
        "store": {
          "vector": {
            "enabled": false
          }
        }
      }
    }
  }
}
```
